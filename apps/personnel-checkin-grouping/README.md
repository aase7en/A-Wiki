# Personnel Check-in & Grouping — รพ.อุทัย

Google Apps Script web app สำหรับเช็คอินเจ้าหน้าที่และจัดกลุ่ม ณ จุดลงทะเบียนกิจกรรมของโรงพยาบาลอุทัย ใช้บนแท็บเล็ต/คีออสก์

## Features

- **ค้นหาชื่อ** — กรองรายชื่อบุคลากรแบบ real-time จากฐานข้อมูลที่อัพโหลด
- **เช็คอิน + จัดกลุ่ม** — เลือกชื่อจากรายการ → กรอกชื่อกลุ่ม → บันทึกลงชีต `Export_Groups`
- **อัพโหลด Excel/CSV** — อัพโหลดไฟล์ `.xlsx`, `.xls`, หรือ `.csv` เป็นฐานข้อมูลบุคลากร (ทุกคอลัมน์, REPLACE)
- **เลือกคอลัมน์สำหรับดาวน์โหลด** — ติ๊กเลือกเฉพาะคอลัมน์ที่ต้องการ (เช่น ชื่อ + ตำแหน่ง + หน่วยงาน ไม่เอาเลขบัตร/เงินเดือน)
- **ล้างฐานข้อมูล** — ลบรายชื่อทั้งหมด (เก็บ `Export_Groups` ไว้)
- **ส่งออก CSV** — ดาวน์โหลดประวัติเช็คอิน (`Export_Groups`) หรือดาวน์โหลดบุคลากรเฉพาะคอลัมน์ที่เลือก (`Personnel_DB`)

## Prerequisites

- Google Account ในองค์กร (ไม่ต้องมีสิทธิพิเศษ)
- ไม่ต้องมี spreadsheet ต้นฉบับ — แอปสร้างชีต `Personnel_DB` ให้อัตโนมัติครั้งแรกที่อัพโหลดไฟล์

## Deploy Steps

1. เปิด Google Drive → New → More → Google Apps Script
2. ลบโค้ดเริ่มต้นใน `Code.gs`; วางเนื้อหาจาก `Code.gs` ใน repo นี้
3. Files → `+` → HTML ตั้งชื่อไฟล์ `Index` (ไม่มีนามสกุล); วางเนื้อหาจาก `Index.html`
4. บันทึก (Ctrl+S); รันฟังก์ชัน `doGet` หนึ่งครั้งเพื่ออนุมัติสิทธิ์ (scope: spreadsheets)
5. Deploy → New deployment:
   - Type: **Web app**
   - Execute as: **Me** (your account)
   - Who has access: **Anyone within [your organization]**
   - Deploy → คัดลอก `/exec` URL
6. เปิด URL บนแท็บเล็ต → กด **อัพโหลด Excel/CSV** เพื่อนำเข้าข้อมูล → เริ่มใช้งาน

## Usage

### ครั้งแรก
1. กด **อัพโหลด Excel/CSV** → เลือกไฟล์ที่มีคอลัมน์ชื่อ (ระบบตรวจจับ `ชื่อ-นามสกุล`, `ชื่อ นามสกุล`, `ชื่อ` หรือ `name` อัตโนมัติ; ถ้าไม่เจอใช้คอลัมน์แรก)
2. ระบบแสดงรายชื่อในช่องค้นหา พร้อมจำนวนบุคลากรและคอลัมน์ทั้งหมด

### เช็คอินประจำวัน
3. พิมพ์ชื่อในช่องค้นหา → ติ๊ก checkbox เพื่อเลือกผู้เช็คอิน
4. กรอกชื่อกลุ่ม (เช่น "กลุ่ม A", "อบรมอัคคีภัย") → กด **บันทึก**
5. ข้อมูลเช็คอินถูก append ลงชีต `Export_Groups` (สร้างอัตโนมัติครั้งแรก)

### Download Personnel_DB (เฉพาะคอลัมน์ที่เลือก)
6. เปิด **เลือกคอลัมน์สำหรับดาวน์โหลด** (คลิกขยาย)
7. ติ๊กเฉพาะคอลัมน์ที่ต้องการ (เช่น ชื่อ, ตำแหน่ง, หน่วยงาน)
8. กด **ดาวน์โหลด (Personnel_DB)** → CSV เฉพาะคอลัมน์ที่เลือก

### Export ประวัติเช็คอิน
9. กด **ดาวน์โหลด CSV (Export_Groups)** เพื่อ export ข้อมูลทั้งหมดจาก `Export_Groups`

### จัดการฐานข้อมูล
- **อัพโหลด Excel/CSV**: REPLACE — เขียนทับข้อมูลเก่าทั้งหมดใน `Personnel_DB` ด้วยไฟล์ใหม่ (ไม่ merge, ไม่มีข้อมูลค้าง)
- **ล้างฐานข้อมูล**: ลบข้อมูลทั้งหมดใน `Personnel_DB`; `Export_Groups` (ประวัติเช็คอิน) ไม่โดนแตะ

### หมายเหตุสำคัญ
- คอนเทนเนอร์-บาวด์สคริปต์ — ถ้าย้าย script ไป spreadsheet อื่น ให้ deploy ใหม่
- รองรับไฟล์ Excel (.xlsx, .xls) และ CSV (.csv) — ใช้ lib SheetJS แปลงฝั่ง browser
- หากเครือข่ายโรงพยาบาลบล็อก CDN (Bootstrap/SheetJS) อาจต้อง host ไฟล์เอง
- ระบบตรวจจับ title row (เช่น "ข้อมูลบุคลากรจำนวน...") อัตโนมัติ — ไม่ต้องลบออกก่อนอัพโหลด

## Sheet Schemas

### Personnel_DB
| Column | Value |
|--------|-------|
| ทุกคอลัมน์จากไฟล์ที่อัพโหลด | ข้อมูลบุคลากรทั้งหมด (เช่น ชื่อ, ตำแหน่ง, หน่วยงาน, เลขบัตร, เงินเดือน, ที่อยู่) |

### Export_Groups
| Column | Value |
|--------|-------|
| Timestamp | วันที่/เวลาที่บันทึก |
| Name | ชื่อเจ้าหน้าที่ |
| Group Name | ชื่อกลุ่มที่ระบุ |
| Status | `Checked-in` |

## Troubleshooting

- **"ยังไม่มีรายชื่อในฐานข้อมูล"**: ยังไม่ได้อัพโหลดไฟล์ — กดปุ่ม "อัพโหลด Excel/CSV"
- **"ไม่พบข้อมูลบุคลากรในไฟล์"**: ระบบหา header หรือคอลัมน์แรกไม่เจอข้อมูล — ตรวจสอบว่าไฟล์มีข้อมูลในคอลัมน์แรก
- **"กรุณาเลือกอย่างน้อย 1 คอลัมน์"**: ก่อนดาวน์โหลด Personnel_DB ต้องติ๊กคอลัมน์ที่ต้องการอย่างน้อย 1 รายการ
- **CSV Export_Groups ว่างเปล่า / มีเฉพาะ header**: ยังไม่มีการบันทึกกลุ่ม — ลองบันทึกกลุ่มก่อนแล้วดาวน์โหลดใหม่
- **หน้าเว็บไม่โหลด/ค้าง**: ตรวจสอบว่า Deploy แล้ว (ไม่ใช่แค่ Save) และตั้ง "Execute as Me" + "Anyone within [org]"
- **Bootstrap UI ไม่โหลด**: เครือข่ายโรงพยาบาลอาจบล็อก CDN — host ไฟล์ Bootstrap และ SheetJS เอง หรือใช้ CSS inline สำรอง
- **ล้างฐานข้อมูลแล้วชื่อยังแสดง**: รีเฟรชหน้า — ระบบจะโหลดใหม่
