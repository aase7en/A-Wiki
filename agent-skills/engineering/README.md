## IRON LAWS Applicable to ALL Engineering Work

- ✅ **Strict TDD:** No production code without a failing test first.
- ✅ **Root Cause First:** No bug fixing without proper investigation (Reproduce → Trace → Falsify → Cross-reference).
- ✅ **No Superficial Patching:** If a parallel model attempts to patch without root cause, discard and rewrite.
