# Patch 16.8 — Affordance Polish

**Status**: ✅ Applied + committed (`chunk(16.8-16.9)` on `main`)
**Cost tier**: L0 (pure store wiring, no paid model)
**Files changed**: `src/state/store.ts`

---

## Goal

Wire the daily activity counters, festival detection, and analyst desk flag so the
Quest Journal reads real data instead of always showing 0. Also fixes the settlement
loop so stakes without a live bot still settle at sleep.

---

## Changes applied

### 1. Farm activity counters → store actions

| Action | Field incremented |
|---|---|
| `seedFarmPlot` | `plotsSeeded += 1` |
| `waterFarmPlot` | `plotsWatered += 1` |
| `harvestFarmPlot` | `plotsHarvested += 1` |
| `shipItem` | `coinsShipped += sellPriceOf(id, day) * amount` |

`sellPriceOf` imported from `logic/market.ts` (same function `sleep.ts` uses for bin
settlement — deterministic, no network).

### 2. `analystDeskOpened` via `setOpenModal`

```typescript
setOpenModal: (m) => set((st) => ({
  openModal: m,
  analystDeskOpened: st.analystDeskOpened || m === 'analyst-desk',
})),
```

Latches to `true` on open; resets to `false` at sleep.

### 3. `isFestivalDay` auto-detected each sleep

```typescript
const nextIsFestivalDay = festivalForDay(result.gameClock.day) !== null
// ... set({ ..., isFestivalDay: nextIsFestivalDay })
```

Uses existing `festivalForDay()` from `logic/festivals.ts` — already imported in store.

### 4. Bot settlement loop fix

Old: iterates `state.bots` → misses stakes without a live configured bot.
New: iterates `state.botStakes` → settles ALL stakes; uses `state.bots[id]?.equityCurve`
for pnlPct, defaults to 0 if bot not configured.
