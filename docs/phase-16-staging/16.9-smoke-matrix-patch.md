# Patch 16.9 — Playable Smoke Matrix

**Status**: ✅ Applied + committed (`chunk(16.8-16.9)` on `main`)

---

## Goal

End-to-end integration tests that verify the full economy loop works from the player's
perspective. These tests exercise store actions in realistic sequences — not just unit
logic — so regressions in wiring are caught immediately.

---

## Tests (12 total, SM-1 → SM-12)

| ID | Flow | Verifies |
|---|---|---|
| SM-1 | Day 1 onboarding | coins=200, botStakes={}, netWorth=0 before sleep |
| SM-2 | Stake → sleep → settled | staked→0 after sleep, netWorthHistory grows |
| SM-3 | Debt payoff partial | balance decrements, coins deducted |
| SM-4 | Debt payoff full | liability removed from list, exact balance spent |
| SM-5 | Emergency buffer set | setEmergencyFundBuffer persists |
| SM-6 | Farm counter reset at sleep | all counters 0 / false after sleep |
| SM-7 | netWorth reflects state | >0 with no debt, matches netWorthHistory[0] |
| SM-8 | Milestone fires + persists | `milestone-10k-networth` in Set after sleep; no re-fire |
| SM-9 | analystDeskOpened flag | set via `setOpenModal('analyst-desk')`, reset at sleep |
| SM-10 | isFestivalDay auto-detect | boolean, day advances after sleep |
| SM-11 | Unstake before sleep | coins returned immediately, stake cleared |
| SM-12 | History cap | ≤30 entries after 5 sleeps |

---

## Notes

- Each test calls `localStorage.clear()` via `beforeEach` (shared localStorage in
  jsdom would bleed between tests otherwise).
- Stakes for SM-2 use a bare `stakeBot('dev-A', 200)` without a configured live bot —
  this verifies the settlement loop iterates `botStakes`, not `bots`.
