# /plan — Break spec into verifiable tasks

Maps to: `skills/engineering-lifecycle/plan/planning-and-task-breakdown/SKILL.md`

## When to use
- You have a validated spec and need implementable units
- Before starting any multi-file implementation

## Flow
1. Run `planning-and-task-breakdown` skill
2. Decompose spec into small, verifiable tasks with acceptance criteria
3. Each task: completable in one session, ≤5 files, has verification step
4. Human reviews task list before advancing to `/build`
