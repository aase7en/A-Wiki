# /spec — Define what to build

Maps to: `skills/engineering-lifecycle/define/spec-driven-development/SKILL.md`

## When to use
- Starting a new project or feature
- Requirements are ambiguous or incomplete
- The change touches multiple files or modules

## Flow
1. Run `spec-driven-development` skill
2. Write spec covering: Objective, Commands, Structure, Code Style, Testing, Boundaries
3. Human reviews and approves before advancing to `/plan`
