## เมื่อข้อมูลมีชีวิต: สร้าง Dashboard สุดล้ำด้วย Node-RED ได้ง่ายๆ ในไม่กี่คลิก!

หลังจากที่เราได้ทำความเข้าใจแล้วว่า **Node-RED** สามารถ `รับ-ส่ง` ข้อมูลจากเซนเซอร์ได้อย่างไรในบทที่แล้ว, คำถามต่อไปที่โคตรจะสำคัญคือ แล้วเราจะ `แสดงผล` ข้อมูลเหล่านั้นได้อย่างไรให้ดูง่ายและเข้าใจได้ในพริบตา? การแสดงผลข้อมูลคือหัวใจของการทำ **Automation** เพราะมันทำให้เราสามารถ `มอนิเตอร์` และ `ตัดสินใจ` ได้แบบ **Real-time** และนั่นคือเหตุผลที่ **Node-RED** มี **Node-RED Dashboard**!

**Node-RED Dashboard** เป็น `ส่วนเสริม (Add-on)` ที่โคตรจะทรงพลัง. มันคือ `เครื่องมือ Low-Code` ที่ช่วยให้คุณสามารถสร้าง **Dashboard** หรือหน้าจอ **UI** ที่สวยงามและตอบสนองได้ (Responsive) โดยไม่ต้องเขียนโค้ด **HTML, CSS, JavaScript** เลยแม้แต่บรรทัดเดียว! มันเหมือนกับมี `ห้องสตูดิโอ` สำหรับการสร้าง **Dashboard** เป็นของตัวเอง

คอนเซ็ปต์การสร้าง **Dashboard** ใน **Node-RED** นั้นโคตรจะเข้าใจง่าย:

1. **ติดตั้ง Nodes ที่จำเป็น:** เริ่มต้นด้วยการติดตั้งแพ็กเกจ `node-red-dashboard` จาก **Manage Palette**
2. **ลาก-วาง UI Nodes:** ในหมวดหมู่ **Dashboard** คุณจะพบกับ **UI Nodes** ที่หลากหลาย ไม่ว่าจะเป็น **Chart**, **Gauge**, **Text**, หรือแม้กระทั่ง **Button** และ **Switch**. คุณเพียงแค่ลาก **Node** ที่ต้องการมาวางบนหน้าจอ **Flow**
3. **เชื่อมต่อกับ Data Flow:** เชื่อมต่อ **UI Nodes** เข้ากับ **Flow** ที่รับข้อมูลจากเซนเซอร์ของคุณ
4. **จัด Group และ Tab:** จัดระเบียบ **Nodes** เหล่านั้นให้อยู่ใน `Group` และ `Tab` เพื่อให้ **Dashboard** ของคุณเป็นระเบียบและดูง่าย

เมื่อคุณทำทุกอย่างเสร็จแล้ว, เพียงแค่กดปุ่ม **"Deploy"** **Node-RED** ก็จะสร้าง **Dashboard** ที่คุณออกแบบให้พร้อมใช้งานได้ทันที. คุณสามารถเข้าถึง **Dashboard** นี้ได้จากทุกอุปกรณ์ที่เชื่อมต่อในเครือข่าย ไม่ว่าจะเป็นคอมพิวเตอร์, แท็บเล็ต หรือมือถือ!

ที่เจ๋งไปกว่านั้นคือ **Node-RED Dashboard** ไม่ใช่แค่แสดงผลข้อมูลอย่างเดียว แต่ยังสามารถเป็น `เครื่องมือควบคุม` ได้ด้วย! คุณสามารถใช้ **Button** หรือ **Switch** บน **Dashboard** เพื่อ `ส่งคำสั่ง` ไปยังอุปกรณ์ต่างๆ เช่น สั่งเปิด-ปิดไฟ, สั่งให้ปั๊มน้ำทำงาน หรือแม้แต่สั่งให้หุ่นยนต์ทำงาน. มันคือการสร้าง **HMI (Human-Machine Interface)** หรือ **SCADA** ขนาดเล็กในแบบของคุณเอง

## UI Nodes: ส่องขุมทรัพย์ Nodes สำหรับ Dashboard ที่โคตรจะครบครัน!

ถ้า **Flow** คือ `ตรรกะ` ของระบบ, **UI Nodes** ก็คือ `หน้าตา` ของระบบของคุณ. การเลือกใช้ **Nodes** ที่เหมาะสมจะช่วยให้ **Dashboard** ของคุณไม่เพียงแต่ดูสวย แต่ยังใช้งานได้จริงและตอบโจทย์ผู้ใช้ ลองมาดูประเภทของ **UI Nodes** ยอดฮิตที่ควรมีไว้ติดตัว:

- **Display Nodes (โหนดแสดงผล):**
	- **UI Gauge:** ใช้แสดงผลข้อมูลที่เป็นตัวเลขในรูปแบบ `มาตรวัด`, เหมาะสำหรับค่าต่างๆ เช่น อุณหภูมิ, ความชื้น, หรือความดัน
		- **UI Chart:** โคตรจำเป็นสำหรับการแสดงผลข้อมูลแบบ `กราฟ`. รองรับทั้ง **Line Chart**, **Bar Chart**, **Pie Chart** ทำให้คุณเห็นแนวโน้มของข้อมูลได้ชัดเจน
		- **UI Text:** ใช้สำหรับแสดงผลข้อมูลที่เป็น `ข้อความ` หรือ `ตัวเลข` แบบง่ายๆ
		- **UI Table:** เหมาะสำหรับการแสดงข้อมูลที่มาในรูปแบบ `ตาราง` ทำให้ดูง่ายและเป็นระเบียบ
- **Input Nodes (โหนดรับคำสั่ง):**
	- **UI Button:** ใช้สำหรับ `สั่งงาน` หรือ `เรียกใช้` **Flow** เมื่อมีการคลิก
		- **UI Switch:** เหมาะสำหรับสั่งงานแบบ `เปิด/ปิด` เช่น เปิด-ปิดไฟ หรือเปิด-ปิดปั๊มน้ำ
		- **UI Slider:** ใช้สำหรับ `ปรับค่า` ตัวเลข เช่น ปรับความเร็วของมอเตอร์ หรือปรับค่าความสว่างของหลอดไฟ

**วิธีการใช้งานที่โคตรง่าย:**

```markdown
[ Sensor ] --(Data Flow)--> [ UI Gauge ]
             |
             '-> [ UI Chart ]
```

Diagram นี้แสดงให้เห็นว่า ข้อมูลจากเซนเซอร์จะไหลไปพร้อมๆ กันเพื่อแสดงผลบน **UI Gauge** และ **UI Chart**. มันคือความยืดหยุ่นที่ **Node-RED** มอบให้!

การปรับแต่ง **Nodes** ก็ทำได้ง่ายๆ แค่ `ดับเบิลคลิก` ที่ **Node** เพื่อเข้าสู่หน้าจอตั้งค่า. คุณสามารถกำหนด `ชื่อ`, `กลุ่ม`, `หน่วย`, และ `ช่วงค่า` ได้ตามต้องการ. มันคือการสร้าง **UI** ที่ไม่จำเป็นต้องมีสกิลด้านการออกแบบเลยแม้แต่น้อย

## บทสรุป (Summary)

### Low-Code Dashboard: เมื่อทุกคนเป็นนักออกแบบ UI ได้

การมี **Dashboard** ที่สวยงามและใช้งานง่ายคือสิ่งที่ขาดไม่ได้สำหรับระบบ **IoT** และ **Automation** และ **Node-RED Dashboard** คือ `เครื่องมือที่ใช่` ที่จะช่วยให้คุณทำสิ่งนี้ได้โดยไม่ต้องพึ่งพานักพัฒนา **Front-End**

มันช่วยให้การแสดงผลข้อมูลและการควบคุมระบบเป็นเรื่องที่โคตรจะง่ายและมีประสิทธิภาพ. ไม่ว่าจะเป็นการมอนิเตอร์อุณหภูมิในห้อง, ควบคุมไฟในบ้าน, หรือแม้แต่แสดงผลข้อมูลการผลิตในโรงงาน, **Node-RED Dashboard** ก็สามารถตอบโจทย์ได้อย่างครบถ้วน

การใช้ **Node-RED** เพื่อสร้าง **Dashboard** จึงเป็นการลงทุนที่คุ้มค่าและเป็นการ `ยกระดับ` โปรเจกต์ของคุณไปอีกขั้น โดยเฉพาะสำหรับผู้ที่ต้องการ `Prototype` หรือ `Proof of Concept` ได้อย่างรวดเร็ว

## ตัวอย่าง (Examples)

**1\. Dashboard สำหรับ Smart Home:**

- **สิ่งที่ต้องการ:** สร้างหน้าจอเพื่อดูอุณหภูมิและความชื้นในบ้าน และมีปุ่มเปิด-ปิดไฟ
- **Flow:** ใช้ **MQTT Node** เพื่อรับข้อมูลจากเซนเซอร์ และใช้ **UI Gauge**, **UI Text** สำหรับแสดงผล. ใช้ **UI Switch** เพื่อสั่งเปิด-ปิดไฟในห้องนั่งเล่น
- **ผลลัพธ์:** คุณมี **Dashboard** สวยๆ ที่สามารถมอนิเตอร์และควบคุมทุกอย่างได้จากหน้าจอเดียวบนมือถือของคุณ

**2\. Dashboard สำหรับ Smart Office:**

- **สิ่งที่ต้องการ:** สร้างหน้าจอเพื่อมอนิเตอร์จำนวนคนในแต่ละห้องประชุมและอุณหภูมิ
- **Flow:** ใช้ **Node** ที่เชื่อมต่อกับ `เซนเซอร์นับคน` และ `เซนเซอร์อุณหภูมิ`. แสดงผลข้อมูลบน **UI Table** และ **UI Chart**.
- **ผลลัพธ์:** ผู้ดูแลอาคารสามารถดูข้อมูลการใช้ห้องประชุมและสภาพแวดล้อมได้แบบ **Real-time** เพื่อช่วยในการจัดการพลังงานและการจองห้อง

**3\. Dashboard สำหรับโรงงานขนาดเล็ก:**

- **สิ่งที่ต้องการ:** สร้างหน้าจอเพื่อมอนิเตอร์สถานะการทำงานของเครื่องจักร 3 ตัว และแสดงจำนวนชิ้นงานที่ผลิตได้
- **Flow:** ใช้ **Nodes** ที่เชื่อมต่อกับ **PLC** แต่ละตัว. ใช้ **UI Text** เพื่อแสดงสถานะ (On/Off) และ **UI Gauge** เพื่อแสดงจำนวนชิ้นงานที่ผลิตได้
- **ผลลัพธ์:** ผู้จัดการสามารถดูประสิทธิภาพการผลิตได้จาก **Dashboard** ในออฟฟิศได้เลย

## คำถามที่พบบ่อย (FAQs)

1. Node-RED Dashboard คืออะไร?เป็น ส่วนเสริม ที่ช่วยให้คุณสร้าง Dashboard หรือหน้าจอ UI ได้ง่ายๆ
2. ต้องเขียนโค้ดในการสร้าง Dashboard หรือไม่?ไม่จำเป็น, เพราะ Node-RED Dashboard ใช้การ ลาก-วาง Nodes ในการสร้าง
3. Dashboard ที่สร้างขึ้นมาเข้าถึงได้จากที่ไหน?จากเว็บเบราว์เซอร์บนทุกอุปกรณ์ที่อยู่ในเครือข่ายเดียวกัน
4. สามารถใช้ Dashboard ควบคุมอุปกรณ์ได้จริงหรือ?ได้, โดยใช้ UI Nodes ที่เป็น Input เช่น Button หรือ Switch
5. Node-RED Dashboard รองรับกราฟแบบไหนบ้าง?รองรับทั้ง Line Chart, Bar Chart, และ Pie Chart
6. สามารถปรับแต่งหน้าตา Dashboard ได้หรือไม่?ได้, สามารถจัดกลุ่ม (Group), สร้างหน้าใหม่ (Tab) และปรับแต่งสีได้
7. การสร้าง Dashboard ใน Node-RED ยากไหม?ง่ายกว่าการเขียนโค้ดเพื่อสร้าง UI ตั้งแต่ต้นเยอะมาก
8. สามารถใช้ Node-RED Dashboard กับ HMI/SCADA ได้ไหม?ได้, สามารถใช้เป็น HMI หรือ SCADA ขนาดเล็กได้ดีเลย
9. ถ้าต้องการให้ Dashboard เข้าถึงได้จากภายนอกเครือข่ายต้องทำอย่างไร?ต้องมีการตั้งค่า Port Forwarding บนเราเตอร์หรือใช้ VPN เพื่อความปลอดภัย
10. Node-RED Dashboard รองรับการแสดงข้อมูลแบบ Real-time หรือไม่?รองรับอย่างแน่นอน, เพราะการสื่อสารใน Node-RED ส่วนใหญ่เป็นแบบ Event-Driven อยู่แล้ว

## ปิดท้ายบทความ: เพิ่มพลังให้ Node-RED ด้วย EC100 Industrial Computer

หลังจากที่เราได้ทำความเข้าใจถึงพลังของ **Node-RED** ที่สามารถพลิกโฉมทั้งโรงงาน, ฟาร์ม, และเมืองได้อย่างน่าทึ่ง. หากคุณต้องการ `ขยายขีดความสามารถ` และสร้างระบบที่แข็งแกร่ง, เชื่อถือได้, และพร้อมสำหรับการใช้งานจริงในสภาพแวดล้อมที่ท้าทาย, การเลือกใช้ `ฮาร์ดแวร์` ที่เหมาะสมคือสิ่งสำคัญ

ขอแนะนำ **EC100 High-performance Open Source ARM Industrial Computer** คอมพิวเตอร์อุตสาหกรรมประสิทธิภาพสูงที่ออกแบบมาเพื่อทำงานร่วมกับ **Node-RED** และโปรโตคอลอุตสาหกรรมอื่นๆ ได้อย่างราบรื่น. ด้วยความที่มันรองรับโปรโตคอลหลากหลายชนิด ไม่ว่าจะเป็น **Modbus**, **OPC UA**, **EtherNet/IP** และอีกมากมาย, ทำให้ **EC100** เป็น `ตัวเลือกที่โคตรจะยืดหยุ่น` ในการเชื่อมต่อกับอุปกรณ์และเครื่องจักรในโรงงานของคุณนอกจากนี้, ด้วยสถาปัตยกรรมแบบ **Open Source** ทำให้คุณสามารถพัฒนาและปรับแต่ง **Flow** ของ **Node-RED** ได้อย่างเต็มที่ โดยไม่ต้องกังวลเรื่องข้อจำกัดด้านซอฟต์แวร์.

สนใจข้อมูลเพิ่มเติม, ติดต่อ **บริษัท เทคซอร์ส กรุ๊ปส์ จำกัด** ตัวแทนจำหน่ายอย่างเป็นทางการ เพื่อเริ่มต้นสร้างระบบอัจฉริยะในแบบของคุณได้เลย!
